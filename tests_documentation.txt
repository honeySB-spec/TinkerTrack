========================================================================
TinkerTrack Test Suite & Verification Documentation
========================================================================

TinkerTrack comes equipped with an automated integration test suite to verify
the core consistency guarantees: Concurrency Control (double-booking prevention)
and Session Security (JWT authentication and salted PBKDF2 cryptography).

Both tests automatically spin up a temporary server instance on port 5001,
run the test scenarios using Node's native fetch API, assert expectations,
and clean up background processes upon completion.

------------------------------------------------------------------------
Test 1: Concurrency Control Verification (test-concurrency.js)
------------------------------------------------------------------------

* File Link: test-concurrency.js
* Purpose: 
  Simulates extreme concurrency by sending two booking requests for the 
  exact same resource and timeslot at the exact same millisecond. It asserts
  that Layer 1 (in-memory Mutex Lock) and Layer 3 (JS overlap constraint)
  successfully block one request with a '409 Conflict' while allowing the
  other request to succeed with '201 Created'.

* How to Run:
  node test-concurrency.js

* Expected Output Log:
  --------------------------------------------------
  Database file not found. Seeding initial data...
  Database seeded successfully!
  Starting temporary test server on port 5001...

  --- Starting Concurrency Test ---
  Simulating 2 users booking the same slot at the exact same millisecond...
  Sending concurrent requests to http://localhost:5001/api/reservations ...

    id: 1,

    status: 'Confirmed'

  Response 2 (Status 409): {
    error: 'Reservation conflict: The selected timeslot overlaps with an existing booking.'

------------------------------------------------------------------------
Test 2: JWT Authentication & Hashing Verification (test-auth.js)
------------------------------------------------------------------------

* File Link: test-auth.js
* Purpose:
  Verifies the cryptographic security pipeline:
  1. Requests to protected endpoints without an Authorization header are blocked (401).
  2. Logins with seeded credentials (encrypted check) succeed and yield valid JWTs.
  3. Logins with incorrect passwords fail (401).
  4. Requests made with a valid JWT token succeed (200 OK).

* How to Run:
  node test-auth.js

* Expected Output Log:
  --------------------------------------------------
  Database file not found. Seeding initial data...
  Database seeded successfully!
  Starting temporary server for Authentication and JWT verification...

  --- Starting JWT Authentication & Encryption Tests ---
  Test 1: Requesting reservations without Authorization header...
  🟢 Passed: Request correctly blocked with 401 Unauthorized.

  Test 2: Logging in as Alice using secure hashed credentials...
  🟢 Passed: Secure login successful. Signed JWT received: eyJhbGciOiJIUzI1NiIsInR5cCI6Ik...

  Test 3: Attempting login with incorrect password...
  🟢 Passed: Login blocked with 401 for incorrect credentials.

  Test 4: Requesting reservations with valid JWT Bearer token...
  🟢 Passed: Request succeeded with 200 OK. Loaded 0 reservations.

  🟢 ALL AUTHENTICATION TESTS PASSED SUCCESSFUL!
  Cleaning up test server...
  --------------------------------------------------
